# 👤 [Story 5] 用户管理与权限模块

> **Labels**: `story` `module: user-management` `priority: medium` `status: planning`
> **Priority**: 🟡 Medium
> **Epic**: [EPIC-social-media-platform.md](./EPIC-social-media-platform.md)
> **Milestone**: M3 — Sprint 5–6
> **Depends On**: Story 4（可视化模块，基于完整 UI 框架集成）

---

## 🎯 业务目标

为平台提供**多租户、基于角色的访问控制（RBAC）**能力，支持不同企业客户（租户）间的数据严格隔离，以及同一企业内不同角色用户的精细化权限管理。同时提供完整的账户生命周期管理功能，支持系统管理员对用户、角色和资源配额进行统一管控，确保平台数据安全和商业模式（按席位/配额计费）的落地基础。

---

## 👥 目标用户

- **平台超级管理员（SaaS 运营方）**：管理所有租户、套餐配额、平台级别配置
- **企业管理员（租户管理员）**：管理本企业内的用户账号、角色分配、资源用量
- **普通业务用户**：登录使用平台功能，管理个人账户设置
- **数据分析师**：在授权范围内查看数据，导出报告

---

## 📖 背景与动机

平台面向多家企业客户提供 SaaS 服务，需要：
1. **数据隔离**：A 公司绝对不能看到 B 公司的监控数据和报告
2. **权限分级**：同一公司内，普通员工不能修改采集任务配置，只有管理员可以
3. **配额管控**：不同套餐的客户有不同的采集量、用户数、存储空间限制
4. **审计追踪**：所有用户操作需可追溯，满足企业安全合规要求

## ✅ 验收标准

### AC-1：用户认证
- [ ] **Given** 用户访问登录页, **When** 输入有效的企业邮箱 + 密码, **Then** 通过认证后跳转至首页，登录响应时间 ≤ 2 秒，同时记录登录日志（时间、IP、设备信息）
- [ ] **Given** 用户连续输入错误密码 **5 次**, **When** 第 5 次失败, **Then** 账号自动锁定 30 分钟，向注册邮箱发送账号锁定通知邮件，锁定期间拒绝所有登录请求
- [ ] **Given** 用户勾选"记住我"选项登录, **When** 认证成功, **Then** Session Token 有效期延长至 30 天（默认 8 小时），关闭浏览器后重新打开无需再次登录
- [ ] **Given** 企业管理员开启了双因素认证（2FA）要求, **When** 员工登录, **Then** 输入密码后需额外验证 TOTP（如 Google Authenticator）或手机短信验证码，两步均通过后方可登录
- [ ] **Given** 用户在 30 分钟内无任何操作, **When** 下次发起请求, **Then** 自动跳转登录页，用户当前页面路由被保存，重新登录后恢复至该页面

### AC-2：用户管理（企业管理员）
- [ ] **Given** 企业管理员进入用户管理页面，**When** 点击"邀请用户"，**Then** 可输入邮箱地址（支持批量，最多 50 个）并分配角色，系统发送邀请邮件，邀请链接 48 小时有效
- [ ] **Given** 被邀请用户点击邮件中的链接，**When** 完成注册（设置密码），**Then** 账号激活，以指定角色加入企业，同时管理员收到用户加入通知
- [ ] **Given** 企业管理员需要禁用某用户，**When** 点击"禁用账号"，**Then** 该用户立即无法登录（已登录 Session 在 5 分钟内失效），用户数据保留，可随时重新启用
- [ ] **Given** 企业管理员查看用户列表，**When** 页面加载，**Then** 展示所有用户的：姓名、邮箱、角色、最近登录时间、账号状态（活跃/未激活/已禁用），支持按角色/状态筛选

### AC-3：角色与权限管理（RBAC）
- [ ] **Given** 平台预设以下角色，**When** 用户被分配角色，**Then** 相应权限立即生效：

  | 角色 | 权限范围 |
  |------|----------|
  | **超级管理员**（平台级）| 所有功能，跨租户管理 |
  | **企业管理员** | 本企业所有功能 + 用户管理 + 配置管理 |
  | **高级分析师** | 所有查看 + 自定义视图 + 报告生成 + 数据导出 |
  | **分析师** | 查看 Dashboard + 话题详情 + 导出（有量限制） |
  | **只读用户** | 查看 Dashboard + 话题列表（不可导出、不可修改任何配置）|

- [ ] **Given** 用户尝试访问超出其角色权限的功能（如只读用户尝试导出数据），**When** 发起请求，**Then** 前端隐藏/禁用相关按钮，后端返回 403，错误信息清晰说明"权限不足，请联系管理员"
- [ ] **Given** 企业管理员需要自定义角色权限，**When** 编辑角色权限矩阵，**Then** 可对每个功能模块的查看/编辑/删除/导出权限单独配置，保存后新权限立即对该角色所有用户生效

### AC-4：多租户数据隔离
- [ ] **Given** 两个不同企业（租户 A 和租户 B）的用户同时登录，**When** 分别查看数据，**Then** 租户 A 的用户只能看到本企业创建的监控任务、采集数据、分析报告，**完全无法**通过任何途径访问租户 B 的数据（包括通过 API、URL 参数修改等方式）
- [ ] **Given** 安全测试人员尝试通过修改 URL 中的资源 ID 访问其他租户的报告，**When** 发起请求，**Then** 后端校验用户所属租户，若资源不属于该租户，返回 404（不返回 403，避免信息泄露），不返回任何他租户数据

### AC-5：资源配额管理
- [ ] **Given** 企业套餐定义了配额（如：最多 10 个用户席位、每日采集量 100 万条、存储 50GB），**When** 企业管理员查看配额页面，**Then** 实时展示已用量 vs 总配额（可视化进度条），并预估按当前使用速度还能使用多久
- [ ] **Given** 用户数量达到套餐席位上限（如已有 10 个用户），**When** 管理员尝试邀请第 11 个用户，**Then** 系统拒绝操作，提示"用户席位已达上限，请升级套餐"，并提供升级引导链接
- [ ] **Given** 企业采集数据量达到当日配额 90%，**When** 超过阈值，**Then** 向企业管理员发送预警邮件，提示即将到达配额上限；达到 100% 时暂停新增采集任务，已有任务继续完成当前批次

### AC-6：个人账户管理
- [ ] **Given** 用户进入个人设置页，**When** 修改个人信息（昵称、头像、通知偏好），**Then** 修改实时保存，页面刷新后变更持久化
- [ ] **Given** 用户想修改登录密码，**When** 提交新密码（需同时输入旧密码），**Then** 密码更新成功，所有其他设备的 Session 立即失效，需重新登录；向注册邮箱发送密码修改通知
- [ ] **Given** 用户忘记密码，**When** 提交密码重置请求（输入邮箱），**Then** 收到重置邮件（5 分钟内），链接有效期 1 小时，点击后可设置新密码，链接使用后立即失效

### AC-7：操作审计日志
- [ ] **Given** 用户在平台执行任何写操作（创建/修改/删除/导出），**When** 操作完成，**Then** 自动记录审计日志（字段：操作时间、操作用户、操作类型、操作对象、操作结果、IP 地址），日志不可修改，保留 180 天
- [ ] **Given** 企业管理员需要审查操作记录，**When** 进入审计日志页面，**Then** 支持按用户、操作类型、时间范围筛选，支持导出为 CSV，响应时间 ≤ 3 秒

## 📌 功能范围

### ✅ In Scope（本期包含）

- 邮箱 + 密码登录认证（含账号锁定机制）
- JWT / Session Token 管理（含"记住我"功能）
- 可选 2FA（TOTP + 短信）
- 企业管理员：邀请用户、禁用/启用用户、角色分配
- 预设 5 种角色（超级管理员/企业管理员/高级分析师/分析师/只读用户）
- 自定义角色权限矩阵
- 严格的多租户数据隔离（数据库级别）
- 资源配额展示 + 配额超限拦截
- 配额预警邮件通知
- 个人账户设置（密码修改、通知偏好）
- 密码重置（邮件链接）
- 操作审计日志（保留 180 天）

### ❌ Out of Scope（本期不包含）

- 第三方 OAuth 登录（企业微信/钉钉/飞书，规划至后续版本）
- 单点登录（SSO/SAML/OIDC 对接企业 IdP，规划至 M4）
- 微信扫码登录
- 硬件安全密钥（FIDO2/WebAuthn）
- 用户行为分析（对平台内部用户的使用行为统计，规划至运营后台）
- 自动化计费和在线支付集成（规划至商业化模块）
- 跨企业协作（跨租户数据共享，暂不规划）

## 🔗 依赖与约束

| 类型 | 内容 |
|------|------|
| **架构依赖** | 所有模块（Story 1–4）均依赖用户认证中间件，但用户管理的完整 UI 在 Story 4 框架上构建 |
| **邮件服务** | 需接入境内邮件发送服务（如阿里云邮件推送、腾讯企业邮），确保邮件送达率 |
| **短信服务** | 2FA 短信需接入境内短信服务商（如阿里云短信、腾讯云短信）|
| **密码策略** | 密码最低强度要求（≥ 8 位，含大小写+数字），前后端双重校验 |
| **合规约束** | 用户数据（邮箱、IP）须按隐私保护要求处理，告知用户数据使用目的 |
| **安全约束** | 密码须 bcrypt/Argon2 哈希存储，禁止明文；HTTPS 强制；CSRF 防护 |
