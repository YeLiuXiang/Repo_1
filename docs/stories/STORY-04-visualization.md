# 📊 [Story 4] 数据可视化与报告导出模块

> **Labels**: `story` `module: visualization` `priority: high` `status: planning`
> **Priority**: 🟠 High
> **Epic**: [EPIC-social-media-platform.md](./EPIC-social-media-platform.md)
> **Milestone**: M2–M3 — Sprint 4–6
> **Depends On**: Story 2（数据存储模块）、Story 3（AI 分析模块）

---

## 🎯 业务目标

将 AI 分析结果通过**直观的可视化界面**和**专业的报告**传递给非技术型业务用户（品牌方、营销团队、管理层），让数据洞察真正落地为业务决策。核心目标：用户从登录到获得核心洞察不超过 **3 分钟**，报告生成时间不超过 **60 秒**，NPS 评分 ≥ 40。

---

## 👥 目标用户

- **品牌方市场负责人**：每日查看品牌舆情 Dashboard，监控关键指标变化
- **营销运营团队**：查看热点话题趋势，发现内容创作机会
- **数据分析师**：使用自定义分析视图，探索深度数据
- **企业管理层**：接收定期推送的 PDF/PPT 分析报告

---

## 📖 背景与动机

数据分析结果如果只以 API 或原始表格形式呈现，非技术用户无法有效使用。可视化模块是平台"最后一公里"交付，决定了用户对平台价值的直接感知。调研显示，83% 的品牌营销决策者更倾向于通过可视化图表和报告获取洞察，而非阅读原始数据。

## ✅ 验收标准

### AC-1：核心 Dashboard（数据看板）
- [ ] **Given** 用户登录平台，**When** 进入首页 Dashboard，**Then** 在 3 秒内加载完成并展示以下核心模块：
  - **舆情概览**：选定关键词在过去 24 小时的情感分布（正/负/中性占比饼图）+ 情感趋势折线图
  - **热点话题**：当前 Top 10 热点话题卡片（含话题名称、热度分、趋势箭头）
  - **平台声量对比**：各平台（微博/抖音/小红书/B站/微信）的数据量对比柱状图
  - **实时数据流**：最新采集内容的滚动列表（可配置显示条数）
- [ ] **Given** 用户切换时间维度（过去 1小时/24小时/7天/30天），**When** 点击时间选择器，**Then** 所有图表在 2 秒内刷新完毕，数据与时间维度保持一致
- [ ] **Given** 负面舆情告警触发，**When** 用户查看 Dashboard，**Then** 告警以醒目红色横幅/弹窗形式展示，包含：告警触发原因、影响关键词、当前负面占比、快速查看链接

### AC-2：话题深度分析页
- [ ] **Given** 用户从热点列表点击某话题，**When** 进入话题详情页，**Then** 展示：
  - 话题传播时间线（横坐标时间、纵坐标声量的面积图）
  - 平台分布热力图（各平台声量随时间变化）
  - 代表性内容展示（Top 10 高互动帖子，含原文摘要、平台来源、互动数据）
  - 相关话题标签云（按共现频率排列）
  - 情感趋势拆分图（正/负/中在时间轴上的堆叠图）
- [ ] **Given** 用户查看话题代表性内容，**When** 点击某条帖子，**Then** 展示完整采集内容（在平台允许展示范围内），并提供跳转原始平台链接
- [ ] **Given** 用户需要比较两个话题，**When** 选择话题对比模式，**Then** 两个话题的关键指标（声量、情感、平台分布）并排展示，支持同时对比最多 3 个话题

### AC-3：自定义分析视图
- [ ] **Given** 数据分析师需要自定义查询，**When** 使用高级筛选器（平台、时间范围、关键词、情感类型、数据量阈值组合），**Then** 图表实时响应筛选条件，刷新时间 ≤ 3 秒
- [ ] **Given** 分析师创建了一组常用筛选条件，**When** 点击保存为"自定义视图"，**Then** 该视图被保存并出现在视图列表中，下次登录后可一键恢复该查询配置
- [ ] **Given** 分析师需要导出原始查询数据，**When** 点击数据导出（CSV/Excel），**Then** 在 30 秒内生成下载文件，文件包含所有展示字段，单次导出上限 10 万条

### AC-4：自动化报告生成
- [ ] **Given** 用户点击"生成报告", **When** 选择报告类型（日报/周报/月报）、关键词范围、格式（PDF/PPT）, **Then** 在 60 秒内完成报告生成，可预览并下载
- [ ] **Given** 管理员配置了定时报告任务（如每周一早 9:00 发送周报）, **When** 到达预定时间, **Then** 系统自动生成报告并通过邮件发送给指定收件人，邮件含报告摘要和 PDF 附件
- [ ] **Given** 生成 PDF 报告, **When** 报告内容包含 AI 分析结论, **Then** 报告须包含免责声明（"本报告由 AI 自动生成，仅供参考，请结合人工判断使用"），且分析数据的统计时间范围须明确标注
- [ ] **Given** 报告模板由管理员配置, **When** 编辑报告模板（logo、色彩方案、包含模块选择）, **Then** 之后生成的所有报告自动应用新模板，已生成报告不受影响

### AC-5：响应式界面与可用性
- [ ] **Given** 用户使用 PC 浏览器（Chrome/Firefox/Edge 最新版）访问平台，**When** 加载各页面，**Then** 所有核心功能正常显示，无功能性 Bug，首屏加载时间 ≤ 3 秒
- [ ] **Given** 用户使用移动端（iOS/Android 主流机型）访问，**When** 浏览 Dashboard，**Then** 界面自适应移动端屏幕，核心图表（舆情概览、热点列表）可正常查看，支持基础交互（切换时间、查看详情）
- [ ] **Given** 图表展示中文内容，**When** 内容包含长文本（如话题名称超过 10 字），**Then** 文本不出现截断或溢出，使用省略号 + Tooltip 完整显示
- [ ] **Given** 用户长时间未操作（30 分钟），**When** 重新操作，**Then** 自动触发重新登录，不丢失当前页面状态（路由保持）

## 📌 功能范围

### ✅ In Scope（本期包含）

- 首页 Dashboard（舆情概览、热点话题、平台声量、实时流）
- 负面舆情告警弹窗/横幅展示
- 话题深度分析页（传播时间线、平台分布、代表性内容）
- 多话题对比视图（最多 3 个）
- 高级筛选器 + 自定义视图保存
- 数据导出（CSV / Excel，上限 10 万条）
- 报告生成（日报/周报/月报，PDF / PPT 格式）
- 定时报告订阅与邮件推送
- 报告模板自定义（logo、色彩、模块）
- PC 端完整功能 + 移动端基础查看
- 报告免责声明与数据标注

### ❌ Out of Scope（本期不包含）

- 嵌入式图表 / 公开分享链接（White-label 功能，规划至 M4）
- 实时流媒体大屏展示（指挥室大屏模式，规划至 M4）
- Word 格式报告导出（规划至下一版本）
- 图表编辑器（用户自定义图表类型，规划至后续版本）
- 多语言界面（当前仅中文）
- 报告内容的 AI 智能摘要（规划至 M3 后续迭代）
- 微信小程序端（规划至后续版本）

## 🔗 依赖与约束

| 类型 | 内容 |
|------|------|
| **上游依赖** | Story 3（AI 分析模块）提供情感/热点/行为分析结果 |
| **上游依赖** | Story 2（数据存储模块）提供聚合统计查询接口 |
| **前端框架** | 建议使用 Vue 3 或 React + Echarts / AntV，需前端架构团队确认技术栈 |
| **图表库** | 推荐 Apache ECharts（支持中文、商业免费），需确认图表性能基准 |
| **PDF 生成** | 推荐使用 Puppeteer（服务端渲染）或专业报告生成库，需评估并发性能 |
| **性能约束** | Dashboard 核心指标须通过预计算缓存支撑，避免实时查询压垮数据库 |
