# 🕷️ [Story 1] 数据采集模块

> **Labels**: `story` `module: data-collection` `priority: critical` `status: planning` `compliance: required`
> **Priority**: 🔴 Critical
> **Epic**: [EPIC-social-media-platform.md](./EPIC-social-media-platform.md)
> **Milestone**: M1 — Sprint 1–2
> **Depends On**: 无（首个交付模块）
> **Blocked By**: 法务合规评审通过

---

## 🎯 业务目标

为平台提供稳定、高效、合规的多平台社交媒体公开数据采集能力。支持**抖音**（短视频、直播、评论）和**知乎**（问答、文章、回答）两大平台的公开内容自动化采集，采集延迟不超过 30 分钟，为下游 AI 分析模块提供持续的原始数据供给。

---

## 👥 目标用户

- **系统管理员**：配置采集任务、关键词、监控范围
- **数据分析师**：依赖采集数据质量开展后续分析工作
- **平台运营团队**：查看采集状态和数据新鲜度

---

## 📖 背景与动机

当前各社交平台数据分散、接口各异，需要一个统一的采集层对多平台数据进行标准化接入。采集模块是整个平台的数据源头，其稳定性和数据质量直接决定后续所有分析结果的可信度。

**关键技术挑战**：
- 各平台反爬策略不同（频率限制、登录墙、JS 渲染等）
- 数据格式差异大，需统一 schema 标准化
- 需在合规边界内最大化数据覆盖率

---

## ✅ 验收标准

### AC-1：多平台采集覆盖（基础能力）
- [ ] **Given** 管理员配置了抖音关键词/话题监控任务，**When** 任务运行，**Then** 系统能采集到相关公开视频的：标题、描述、话题标签、点赞/评论/分享数、发布时间、发布账号公开信息，**且** 采集成功率 ≥ 95%（连续 7 天统计）

- [ ] **Given** 管理员配置了抖音视频评论采集任务，**When** 任务运行，**Then** 能采集到指定视频下的公开评论（含：评论文本、点赞数、发布时间、评论用户公开昵称），数据完整率 ≥ 90%

- [ ] **Given** 管理员配置了知乎关键词/话题监控任务，**When** 任务运行，**Then** 能采集到该关键词相关的公开问答（含：问题标题、问题描述、回答内容、点赞/评论/收藏数、发布时间、回答者公开信息），采集成功率 ≥ 95%

- [ ] **Given** 管理员配置了知乎专栏文章监控，**When** 任务运行，**Then** 能采集到专栏文章的标题、正文摘要、发布时间、点赞/评论数

- [ ] **Given** 管理员配置了知乎热榜监控，**When** 任务运行（每小时一次），**Then** 能采集到当前热榜 Top 50 问题的标题、热度分、浏览量

### AC-2：采集任务调度与配置
- [ ] **Given** 管理员在控制台创建采集任务，**When** 提交配置，**Then** 任务配置项包括：目标平台（多选）、关键词列表（支持布尔运算符）、采集频率（实时/小时/日）、数据范围（时间窗口）、存储目标
- [ ] **Given** 采集任务已启动，**When** 管理员查看任务列表，**Then** 能实时看到每个任务的状态（运行中/暂停/失败/已完成）、已采集条数、最近采集时间、错误率
- [ ] **Given** 管理员需要临时暂停某采集任务，**When** 点击暂停，**Then** 任务在 60 秒内停止，已采集数据不丢失，恢复后从断点继续

### AC-3：数据标准化与去重
- [ ] **Given** 采集到来自不同平台的原始数据，**When** 进入数据管道，**Then** 统一转换为标准 Schema（字段：`id`, `platform`, `content`, `author_id`, `publish_time`, `engagement_stats`, `media_urls`, `tags`, `raw_data`），所有时间戳统一转换为 UTC+8
- [ ] **Given** 同一条内容被多次采集（如热门帖子被多个任务覆盖），**When** 写入存储，**Then** 系统基于平台原始 ID 进行去重，重复率 ≤ 2%
- [ ] **Given** 原始采集内容包含特殊字符或 Emoji，**When** 标准化处理，**Then** 内容完整保留，不出现乱码或截断

### AC-4：反爬熔断与容错
- [ ] **Given** 某平台连续 5 次采集请求失败（超时或被封禁），**When** 触发熔断，**Then** 该平台采集任务自动降级（降低频率至 1/10），并向管理员发送告警通知（邮件/企业微信），等待 30 分钟后自动尝试恢复
- [ ] **Given** 网络故障导致采集中断，**When** 网络恢复，**Then** 任务自动重启并补采中断期间的数据（支持基于时间戳的增量补采）
- [ ] **Given** 采集到包含明显违规内容的数据，**When** 内容安全预筛查触发，**Then** 该条数据被标记为 `flagged`，不进入正常分析流程，等待人工审核

### AC-5：采集监控与可观测性
- [ ] **Given** 平台运行中，**When** 管理员访问监控面板，**Then** 能实时查看：各平台采集速率（条/分钟）、队列积压数、错误率、存储用量趋势
- [ ] **Given** 某采集任务错误率超过 10%，**When** 持续超过 15 分钟，**Then** 系统自动创建告警记录，并通知相关负责人
- [ ] **Given** 需要审计采集行为，**When** 查询采集日志，**Then** 每条日志包含：采集时间、目标 URL/接口、响应状态码、数据量、任务 ID，日志保留 90 天

## 📌 功能范围

### ✅ In Scope（本期包含）

- 抖音、知乎两大平台的公开数据采集（抖音：短视频/直播/评论；知乎：问答/专栏文章/热榜）
- 关键词/账号/话题三种监控维度的任务配置
- 统一数据标准化 Schema 及 ETL 管道
- 采集任务调度、暂停、恢复、删除管理
- 基于平台原始 ID 的去重机制
- 反爬熔断降级策略
- 采集监控面板（任务状态 + 性能指标）
- 内容安全预筛查（违规内容标记）
- 采集操作审计日志

### ❌ Out of Scope（本期不包含）

- 私信、私密账号、付费内容的采集（违规，永久不做）
- 抖音直播内容实时流采集（复杂度高，规划至下一期）
- 知乎盐选专栏（付费内容，违规）
- 其他平台（微博、B站、小红书、微信公众号等，规划至后续版本）
- 境外平台（Twitter/X、Instagram、TikTok 国际版）
- 图片/视频媒体文件的本地存储（仅存储 URL）
- 基于用户身份的个性化采集（如登录态操作）

## 🔗 依赖与约束

| 类型 | 内容 |
|------|------|
| **前置依赖** | 法务团队完成各平台采集合规评审，形成书面结论 |
| **系统依赖** | 数据存储模块（Story 2）提供写入接口 |
| **外部依赖** | 抖音开放平台 API 申请、知乎 API 评估（建议优先使用官方接口，降低风险）|
| **基础设施** | 境内云服务器（出口 IP 池）、代理 IP 服务（合规前提下） |
| **技术约束** | 采集频率须符合各平台 robots.txt 要求，不得对目标平台造成负载压力 |
| **安全约束** | 采集账号凭据（如 API Key）须加密存储，不得明文出现在代码或日志中 |
